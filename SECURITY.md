# Security

This project is maintained by volunteers for the **[Toronto Product Management Association (TPMA)](https://www.tpma.ca/)**.

## Reporting a vulnerability

**Do not** open a public issue, discussion, or PR that describes how to exploit the problem before maintainers have assessed it.

**Preferred:** If this repository has **[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)** enabled, use that flow in the **Security** tab.

**Otherwise:** Open a **[GitHub Security Advisory](https://docs.github.com/en/code-security/security-advisories)** against this repository when the feature is available, or email **[info@tpma.ca](mailto:info@tpma.ca)** or **[president@tpma.ca](mailto:president@tpma.ca)** (see **[tpma.ca](https://www.tpma.ca/)** for general association contact options).

Include enough detail to reproduce or understand the impact (affected component, version or commit, reproduction steps if safe to share). Treat any information you receive as confidential unless the maintainers confirm otherwise.

You should receive an acknowledgment within a reasonable time; volunteers may need a few days. We will coordinate a fix and disclosure schedule with you when appropriate.

## Supported versions

Security fixes are aimed at the **default branch** (`main` unless changed) and any release tracks the maintainers document. For production deployments, keep dependencies updated and follow **`README.md`** for environment and hosting practices.
